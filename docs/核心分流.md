# PGH 核心与运行时分流

PGH 分成两层：**PGH Core** 和 **Runtime Adapter**。Core 定义记忆与工作流，Adapter 负责把同一套逻辑接到具体 AI 运行时。

## PGH Core

Core 位于 `assistant/`，不绑定 Claude 或 Codex。

它包含：

- `USER/`：用户稳定档案与子文件。
- `SOUL/persona/`：AI 人格与行为身份。
- `MEMORY/`：v5 默认遗忘记忆架构。
- `00 专注区/`、`01 项目区/`、`02 阅读区/`、`03 写作区/`：长期工作区。
- `长期记忆.md`：当前处境、时间轴与周录。
- `ITERATION_LOG.md`：架构、skill、协议变更。

Core 的关键原则：

- 加载链导航，不靠模型硬记目录。
- 单一权威源，其他位置只放摘要和指针。
- 默认遗忘，低置信号先入 L0/L1，高稳定模式才进入 L2/L3。
- 身份层写入需要用户明确确认。

## Codex Code Adapter

Codex 适配层位于 `.codex/`。

它负责：

- `AGENTS.md`：全局启动序列与行为路由（顶部 §0 节负责首次初始化）。
- `config.toml`：Codex hooks 开关与 hook 命令。
- `hooks/*.py`：逐消息注入记忆信号、思考协议、节点收尾提示和结束复盘提醒。
- `skills/`：节点闭合、日复盘、周复盘、项目推进、新建项目等流程。
- `agents/`：可选的检索或领域 agent。

Codex 端使用 hooks 做运行时提醒和边界提示；长期记忆仍由 `assistant/` 持久化，身份层与 semantic memory 由启动序列读取。

## 分流边界

- Core 文件不写 Codex 专属语义，例如 hook 命令、config.toml 字段、Python 脚本路径。
- Codex Adapter 不重新定义记忆架构，只引用 Core 的 `MEMORY/00.记忆区_agent.md`。
- README 只说明本仓库的运行时；跨运行时共同逻辑放在本文件。
- 本仓库是单端自用模板，不内置跨端通信或跨智能体转交层。
- 私人信息不进入公开模板，初始化后由使用者本地生成。
