# write-progress

当项目工作产生推理、决策、尝试或新问题，需要跨会话保留时调用。

## 推进区结构

```text
<项目>/_progress/
├── 00_progress_index.md
└── YYYY-MM-DD_短主题.md
```

## 推进节点模板

```markdown
---
标题: <节点标题>
类型: 推进记录
创建日期: YYYY-MM-DD
项目: <项目名>
上游:
  - ../_overview.md
同级联动: []
---

# <节点标题>

## 问题承接

## 推进

## 结论 / 当前判断

## 新问题

## 下一步
```

## 流程

1. 定位项目和 `_overview.md`。
2. 缺少 `_progress/` 时创建，并补推进索引。
3. 判断是追加现有推进文件，还是新建节点文件。
4. 记录推理路径，不只写最终答案。
5. 必要时使用转换标记：
   - `**问题承接**`
   - `**开出**`
   - `**本节收束**`
6. `_overview.md` 只更新稳定状态和指针，不塞完整讨论。
7. 用户授权后再更新 `_本周.md`。

## 不做

- 不把稳定项目结论只埋在推进记录里。
- 不把每一句聊天都写成 progress。
- 不在这里写个人身份事实；走记忆复盘和身份层确认。
