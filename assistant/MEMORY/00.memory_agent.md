# Memory Agent

This file owns memory metabolism rules. `AGENTS.md` names the global layers; this file owns local formats.

## Layer Roles

| Layer | File | Load at startup | Default fate |
|---|---|---|---|
| L0 episodic inbox | `episodic_inbox.md` | no | expire unless repeated |
| L1 episodic memory | `episodic_memory.md` | no | decay, merge, or promote |
| L2 semantic memory | `semantic_memory.md` | yes, active entries only | startup schema |
| L3 identity | `USER/`, `SOUL/`, `.codex/skills/` | yes / skill-triggered | stable after approval |

## P/C Axes

- P axis: hit / breakthrough / uncovered
- C axis: confirmation / objection / correction / neutral

Only `P=hit` and `C=neutral` is normally not written. Other combinations may enter L0 or L1 when useful.

## L0 Inbox Row

```markdown
| JYYYYMMDD-001 | YYYY-MM-DD HH:mm | source | P | C | type | schema pointer | one-line signal |
```

## L1 Episodic Entry

```markdown
## EM-YYYYMMDD-001 · Title

- Strength: ★
- Type: preference / procedure / boundary / project-pattern / other
- Status: active
- Created: YYYY-MM-DD
- Last activated: YYYY-MM-DD
- Activation count: 1
- Trigger context:
- Action / expectation:
- Source:
- Related:
- Notes:
```

## L2 Semantic Entry

```markdown
## SM-YYYYMMDD-001 · Title

- Strength: ★★★★
- Type: preference / procedure / boundary / identity-candidate / other
- Status: active
- Created: YYYY-MM-DD
- Last activated: YYYY-MM-DD
- Evidence: `_archive/semantic_archive.md#...`
- Schema:
- Prediction:
- Action:
- Graduation candidate: no
```

## Promotion / Decay

- L0 -> L1: repeated signal, strong correction, or clear uncovered expectation.
- L1 -> L2: stable across contexts and useful for future prediction.
- L2 -> L3: 6 stars, cross-week stability, and explicit user approval.
- Decay: unused, contradicted, or too situational entries should lose strength or archive.

## Archive Rule

Keep main memory files short. Long evidence goes into `_archive/semantic_archive.md` with pointers from active entries.
